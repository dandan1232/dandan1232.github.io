基于 PyTorch 的轻量推荐系统框架 Torch-RecHub

发布于 2026-05-07 · 推荐系统

项目简介

Torch-RecHub 是由 Datawhale 社区维护的、基于 PyTorch 构建的轻量推荐系统框架。它旨在简化推荐算法的研究和应用，提供常见的模型实现、数据处理工具和评估指标。

  • 项目地址：github.com/datawhalechina/torch-rechub
  • 文档地址：datawhalechina.github.io/torch-rechub/

核心特性

  • 模块化设计：易于添加新的模型、数据集和评估指标
  • 基于 PyTorch：利用动态图和 GPU 加速能力
  • 丰富的模型库：包含 30+ 种经典和前沿推荐算法
  • 标准化流程：统一的数据加载、训练和评估流程
  • 易扩展：模型训练与模型定义解耦，无 basemodel 概念
  • 原生函数：尽可能使用 PyTorch 原生类与函数，方便新手学习
  • 生产部署：支持 ONNX 导出，便于部署到生产环境

安装指南

系统要求

  • Python 3.9+
  • PyTorch 1.7+（推荐 CUDA 版本）
  • NumPy、Pandas、SciPy、Scikit-learn

安装方式

# 稳定版（推荐）

# 最新开发版

整体架构

Torch-RecHub 采用分层架构设计：

  • 特征层：处理 DenseFeature（数值型）、SparseFeature（类别型）、SequenceFeature（序列型）
  • 数据层：TorchDataset / PredictDataset / DataGenerator / MatchDataGenerator
  • 模型层：排序模型、召回模型、多任务模型、生成式推荐模型
  • 训练层：CTRTrainer / MatchTrainer / MTLTrainer
  • 工具层：ONNX 导出、模型可视化、回调函数、损失函数

模型库总览（30+ 模型）

排序模型（Ranking）— 13 个

用于精排阶段，预测用户对物品的点击率或偏好分数。

模型适用场景核心特点
Wide&Deep基础排序任务结合线性和深度模型，兼顾记忆和泛化
DeepFM特征交互重要场景同时捕获低阶和高阶特征交互
DCN / DCNv2显式特征交叉显式学习高阶特征交叉，计算效率高
DIN用户兴趣动态变化基于注意力机制捕捉用户兴趣
DIEN长序列兴趣建模建模用户兴趣的动态演化过程
BST序列特征重要场景使用 Transformer 建模序列特征
AutoInt自动特征交互学习自动学习特征交互模式

召回模型（Matching）— 12 个

用于粗排阶段，从海量物品中召回候选集。

模型适用场景核心特点
DSSM文本匹配场景双塔结构，用户和物品映射到同一向量空间
YoutubeDNN大规模推荐基于用户行为序列的深度召回
MIND多兴趣推荐为用户学习多个兴趣表示
GRU4Rec / SASRec序列推荐建模用户近期行为序列
ComiRec可控多兴趣推荐允许控制生成的兴趣数量

多任务模型（Multi-Task）— 5 个

同时优化多个相关任务，提高模型泛化能力。

模型适用场景核心特点
SharedBottom任务相关性强所有任务共享底层网络
MMoE任务冲突较大多门控专家混合，为不同任务学习不同专家组合
PLE复杂多任务渐进式分层提取，缓解负迁移
ESMM样本选择偏差全空间建模，解决样本选择偏差
AITM任务间有依赖自适应信息迁移，学习任务间依赖关系

生成式推荐（Generative）— 2 个

模型适用场景核心特点
HSTU大规模序列推荐层级序列转换单元，支撑万亿参数推荐系统
HLLM融合 LLM 能力结合大语言模型的语义理解能力

快速上手

排序模型示例

from torch_rechub.models.ranking import DeepFM
from torch_rechub.trainers import CTRTrainer

# 创建模型
model = DeepFM(
    deep_features=deep_features,
    fm_features=fm_features,
    mlp_params={"dims": [256, 128], "dropout": 0.2}
)

# 训练
trainer = CTRTrainer(model, optimizer_params={"lr": 0.001}, device="cuda:0")
trainer.fit(train_dataloader, val_dataloader)

召回模型示例

from torch_rechub.models.matching import DSSM
from torch_rechub.trainers import MatchTrainer

# 创建模型
model = DSSM(
    user_features=user_features,
    item_features=item_features,
    temperature=0.02,
    user_params={"dims": [256, 128, 64]},
    item_params={"dims": [256, 128, 64]}
)

# 训练
trainer = MatchTrainer(model, mode=0, device="cuda:0")
trainer.fit(train_dataloader)

模型选择指南

  • 刚入门推荐系统：从 Wide&Deep 和 DSSM 开始，理解排序和召回的基本范式
  • 特征工程重要：优先尝试 DeepFM、DCN，它们对特征交互建模能力强
  • 用户行为序列丰富：DIN、DIEN、SASRec 能有效捕捉时序兴趣
  • 多目标优化：MMoE、PLE 是业界最常用的多任务模型
  • 大模型融合：关注 HLLM，代表了 LLM + 推荐的前沿方向

总结

Torch-RecHub 是一个非常适合学习和实践推荐系统的框架。它的代码精简、模块化设计清晰，模型覆盖了从经典到前沿的各个阶段。无论你是推荐系统初学者还是需要快速验证想法的工程师，都值得深入了解。
