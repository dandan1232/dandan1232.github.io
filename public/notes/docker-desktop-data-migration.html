Docker Desktop 数据迁移笔记：把 WSL2 数据从 C 盘迁到 D 盘

发布于 2026-05-28 · Docker / WSL2 / Windows

问题背景

Docker Desktop 在 Windows 上通常使用 WSL2 作为后端。镜像、容器、卷等数据会逐渐写入本机用户目录下的 Docker 数据文件，尤其是 docker_data.vhdx，很容易占用 C 盘空间。更稳妥的做法是把 Docker 数据目录迁移到空间更充足的 D 盘，再通过符号链接保持原路径可用。

迁移原理

迁移并不是修改业务项目配置，而是调整 Docker Desktop 的本地数据存储位置：

  • 移动数据：把 %LOCALAPPDATA%\Docker 整体移动到 D:\Docker。
  • 建立符号链接：在原路径创建指向新目录的目录链接。
  • 保持透明：Docker Desktop 仍访问原路径，但文件实际落在 D 盘。

数据目录结构

核心数据通常集中在 C:\Users\{用户名}\AppData\Local\Docker：

C:\Users\{用户名}\AppData\Local\Docker\
├── log\                    # 日志文件
├── run\                    # 运行时文件
├── tasks\                  # 任务配置
└── wsl\                    # WSL2 虚拟磁盘
    ├── disk\
    │   └── docker_data.vhdx  # Docker 镜像、容器、卷数据
    └── main\
        └── ext4.vhdx         # WSL2 系统文件

迁移前检查

先确认 Docker 当前状态、数据大小和目标磁盘空间。建议在 PowerShell 中执行：

# 检查 Docker 信息
docker info

# 统计 Docker WSL 数据大小
Get-ChildItem "$env:LOCALAPPDATA\Docker\wsl" -Recurse |
  Measure-Object -Property Length -Sum |
  Select-Object @{Name="SizeGB";Expression={[math]::Round($_.Sum/1GB,2)}}, Count

# 检查 D 盘空间
Get-WmiObject -Class Win32_LogicalDisk |
  Where-Object {$_.DeviceID -eq "D:"} |
  Select-Object DeviceID,
    @{Name="SizeGB";Expression={[math]::Round($_.Size/1GB,2)}},
    @{Name="FreeSpaceGB";Expression={[math]::Round($_.FreeSpace/1GB,2)}}

迁移步骤

1. 停止 Docker 与 WSL

VHDX 文件运行时会被锁定，必须先完整停止 Docker Desktop 和 WSL。

docker desktop stop
wsl --shutdown
Get-Process -Name "*Docker*" -ErrorAction SilentlyContinue

2. 创建目标目录

New-Item -Path "D:\Docker" -ItemType Directory -Force

3. 移动 Docker 数据

推荐使用 robocopy，失败重试能力更好：

robocopy "$env:LOCALAPPDATA\Docker" "D:\Docker" /E /MOVE /R:3 /W:5
  • /E：包含所有子目录，包括空目录。
  • /MOVE：复制完成后删除源目录内容。
  • /R:3：失败后最多重试 3 次。
  • /W:5：每次重试间隔 5 秒。

如果 robocopy 不适合当前环境，也可以先复制再删除：

Copy-Item "$env:LOCALAPPDATA\Docker" "D:\Docker" -Recurse -Force
Remove-Item "$env:LOCALAPPDATA\Docker" -Recurse -Force

4. 创建符号链接

以管理员身份运行 PowerShell：

New-Item -Path "$env:LOCALAPPDATA\Docker" -ItemType SymbolicLink -Value "D:\Docker" -Force

如果 PowerShell 创建失败，可以用管理员 CMD 执行：

mklink /D "C:\Users\{用户名}\AppData\Local\Docker" "D:\Docker"

验证与启动

迁移完成后，先确认符号链接和 D 盘数据，再启动 Docker Desktop。

# 检查符号链接
Get-Item "$env:LOCALAPPDATA\Docker" | Select-Object LinkType, Target

# 检查目标目录
dir "D:\Docker"

# 检查主要数据文件
Get-ChildItem "D:\Docker\wsl\disk" |
  Select-Object Name, @{Name="SizeGB";Expression={[math]::Round($_.Length/1GB,2)}}

# 启动并验证 Docker
docker desktop start
docker info
docker images

常见问题

权限不足

创建符号链接时提示需要管理员权限，优先以管理员身份打开 PowerShell。Windows 开发者模式也可以降低创建符号链接的权限门槛。

文件被占用

如果移动时报 ERROR 32，说明仍有 Docker 相关进程或 WSL 实例占用文件：

Get-Process -Name "*Docker*" | Stop-Process -Force
wsl --shutdown
taskkill /F /IM "Docker Desktop.exe"

符号链接失效

Docker Desktop 启动后找不到数据时，先检查原路径是否确实是符号链接，必要时重建：

Get-Item "$env:LOCALAPPDATA\Docker"

Remove-Item "$env:LOCALAPPDATA\Docker" -Force
New-Item -Path "$env:LOCALAPPDATA\Docker" -ItemType SymbolicLink -Value "D:\Docker" -Force

WSL 启动失败

如果 Docker Desktop 启动时报 WSL 错误，可以先关闭 WSL。只有在确认可以接受 Docker WSL 分发被重建时，再执行 unregister 操作。

wsl --shutdown
wsl --unregister docker-desktop
wsl --unregister docker-desktop-data

回滚方案

如果迁移后表现异常，可以删除符号链接并把数据移回 C 盘：

docker desktop stop
wsl --shutdown

Remove-Item "$env:LOCALAPPDATA\Docker"
robocopy "D:\Docker" "$env:LOCALAPPDATA\Docker" /E /MOVE
Remove-Item "D:\Docker" -Recurse -Force

维护建议

  • 磁盘类型：优先放在 SSD，镜像解压、容器启动和构建速度会更稳定。
  • 空间余量：目标盘建议至少保留 20% 可用空间，避免 VHDX 扩容失败。
  • 定期清理：无用镜像、构建缓存和卷会持续膨胀，需要周期性清理。
  • 备份意识：重要容器卷迁移前应单独备份，避免把 Docker 数据目录当作唯一备份。
docker system prune -a
docker volume prune

迁移后的结果

最终目录关系应当类似下面这样：

C:\Users\{用户名}\AppData\Local\Docker  ->  D:\Docker  # 符号链接

D:\Docker\
├── wsl\disk\docker_data.vhdx
├── wsl\main\ext4.vhdx
└── 其他配置、日志和运行时文件

这套方案的关键是：先停服务，再移动完整目录，最后用符号链接把 Docker Desktop 的默认访问路径接回去。
