FastGPT 知识库结构讲解

整理于 2026-05-07 · 来源：Obsidian 笔记库

本文介绍 FastGPT 知识库的结构设计，理解其 QA 存储格式和多向量映射，以便更好地构建知识库。

理解向量

FastGPT 采用 RAG 中的 Embedding 方案构建知识库。人类的文字、图片、视频等媒介无法直接被计算机理解，需要将它们转成向量（数字数组）。两个向量之间可以通过数学公式得出距离，距离越小代表相似度越大。

向量搜索使用 TopK 召回方式：查找前 K 个最相似的内容，丢给大模型做更进一步的语义判断、逻辑推理和归纳总结。

影响向量搜索精度的因素

  • 向量模型的质量 — 通用模型在特定领域精度有限
  • 数据的质量 — 长度、完整性、多样性
  • 检索器的精度 — 速度与精度之间的取舍
  • 检索词的质量 — 用户问题通常是模糊或缺失的

FastGPT 向量结构设计

FastGPT 采用 PostgreSQL + PGVector 作为向量检索器（HNSW 索引），MongoDB 用于其他数据存取。

MongoDB (dataset.datas 表) ├── 存储向量原数据 ├── indexes 字段 → 记录对应的向量 ID（数组） └── 一组向量可对应多组数据 PostgreSQL (PGVector) ├── index 字段 → 存储向量 └── 检索时：先召回向量 → 根据 ID 去 MongoDB 找原数据 → 同一组原数据则合并，向量得分取最高

多向量映射

内容的长度和语义丰富度通常是矛盾的。FastGPT 采用多向量映射：为一组较长的文本添加多组向量，检索时只要其中一组被命中，该数据就会被召回。兼顾数据完整性和语义丰富度。

提高搜索精度的方法

1. 更好分词分段

当一段话的结构和语义是完整的、单一的，精度会提高。优化分词器，尽可能保障每组数据的完整性。

2. 精简 Index 内容

减少向量内容的长度，检索精度自然提高。但会牺牲检索范围，适合答案较为严格的场景。

3. 丰富 Index 数量

为同一个 Chunk 内容增加多组 Index，提高命中概率。

4. 优化检索词

用户的问题通常是模糊或缺失的。优化用户问题（检索词）很大程度上可以提高精度。

5. 微调向量模型

通用向量模型在特定领域检索精度不高，微调可以大幅提升专业领域的检索效果。

知识库层级结构

库（最小搜索单位） └── 集合（相当于一个文件，用于分类管理） └── 数据（最小存储单元）

搜索时对整个库进行搜索，集合仅用于数据分类管理，与搜索效果无关。

数据导入方案

方案 1 — 直接分段导入

利用句子分词器对文本按一定长度拆分，分割成多组 Q。建议使用通用引用提示词模板。

方案 2 — QA 导入

分词后调用大模型对分段学习，自动生成问题和答案，一起存储到 Q 中。新版 FastGPT 为提高搜索范围，不再将 QA 分开存储。

方案 3 — 手动录入

在集合中点击"手动录入知识点"，或使用标注功能。被搜索的内容为 Q，补充内容为 A。

方案 4 — CSV 录入

对独特数据进行预处理分割后，通过 CSV 批量导入。

方案 5 — API 导入

通过 FastGPT OpenAPI 进行程序化导入，适合自动化流水线。

落地建议

  • 先用直接分段快速验证效果，再根据精度决定是否切换 QA 模式
  • 对专业术语多的场景，优先考虑微调向量模型
  • 多向量映射是提升召回率的低成本手段，建议每个 Chunk 至少 2-3 组 Index
  • 检索词优化（Query Rewrite）往往比调模型更有效
