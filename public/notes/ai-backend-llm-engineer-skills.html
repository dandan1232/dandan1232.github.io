AI 后端 LLM 工程师 — 技能雷达图

整理于 2026-05-07 · 来源：Obsidian 笔记库

本文是一份面向 AI 应用工程师（非算法研究员）的技能分层路线图。核心思路：不学 CNN、不手写反向传播，不是"没学 AI"，而是已经走在"真正赚钱的 AI 工程路线"上。

一、技能雷达总览

AI / 模型相关

技能级别说明
机器学习基础概念该会不推公式，只懂原理
深度学习基础该会重点 Transformer
Transformer 原理会Attention、Embedding
向量 Embedding会RAG 核心
Reranker会检索效果关键
LoRA / 微调该会不必亲自训练
模型训练（从零）不用会算法岗内容
CNN / CV 算法不用会非主线

RAG / 知识工程（主战场）

技能级别说明
文本切分策略会Chunk 质量决定上限
向量检索会pgvector / Milvus
混合检索（BM25 + 向量）会工业级必备
GraphRAG / LightRAG会实体/关系检索
Neo4j 图建模会实体 / 关系
幻觉控制会Prompt + RAG
检索召回评估该会Recall / TopK
知识图谱算法不用会GNN 非必须

Prompt & Agent

技能级别说明
Prompt 设计会System Prompt
Few-shot / CoT会提升稳定性
Tool Calling会Agent 核心
ReAct 框架该会行业通用
多 Agent 协作该会高阶加分

工程能力（最重要）

技能级别说明
Python 工程化会模块化 / 规范
FastAPI会主力框架
异步并发会性能瓶颈点
Docker / Compose会必备
Linux会日常
CI/CD该会加分项
C++ / Rust不用会ROI 极低

数据库 & 存储

技能级别说明
MySQL会业务数据
PostgreSQL / pgvector会向量库
Neo4j会图谱
Redis会缓存
MongoDB该会文档存储

二、投入优先级

必须 100% 投入

  • RAG / GraphRAG
  • Prompt 工程
  • FastAPI + Docker + Nginx
  • 数据库协同（PG + Neo4j）

选修（锦上添花）

  • Agent 工程体系（Tool 调度、Memory 策略、执行控制）
  • 多模态
  • LoRA 原理
  • MQ + 异步 AI 任务架构（Kafka / RabbitMQ）

可以明确放弃

  • 手写神经网络 / 反向传播推导
  • 算法竞赛
  • CV / RL 深度研究
  • 自研深度学习框架

三、学习路径（6 个月规划）

阶段时间内容
基础第 1-2 月Python 基础、数学基础、模型高层次工作原理
ML 入门第 3 月NumPy / Pandas / Matplotlib、机器学习基础、3 个迷你项目
DL + NLP第 4 月PyTorch、小型神经网络、NLP + CV 各 1 个项目
LLM 实战第 5 月LLM 基础、OpenAI / Anthropic API、RAG 应用、向量数据库
项目 + 部署第 6 月3 个真实项目、部署上线、GitHub 作品集、简历 + 面试

四、当前阶段重点补强

Phase 1 — 马上做

  • RAG 工业化：多路召回、Reranker 链路、检索评估体系
  • NumPy / Pandas：工程补齐，不怕看代码、能改就行

Phase 2 — 直接拉开差距

  • Agent 工程体系：Tool 调度、Memory 策略、执行控制
  • MQ + 异步 AI 任务架构：推理任务异步化、批量 embedding 任务队列

Phase 3 — 架构层

  • 数据生命周期管理：增量 embedding、索引重建策略、多租户隔离
  • AI 服务网关：模型路由、限流、token 控制、统一日志
  • 推理成本结构：token 成本、延迟瓶颈、并发极限、cache 收益

五、能力分层定位

用户请求 ↓ SpringBoot / FastAPI ↓ RAG 服务 ↓ 向量数据库 + 图数据库 ↓ LLM ↓ 缓存 / 消息队列 ↓ 高并发系统部署
层级定位特征
L1AI 工具使用者会调 OpenAI、会 LangChain、会搭简单 RAG（大量人停在这里）
L2AI 应用工程师会向量库、会 RAG 优化、会评估指标、会部署、会成本控制
L3AI 系统架构工程师MQ 异步任务、多模型路由、推理成本结构、多租户数据隔离、灰度发布

六、最大误区

  • 直接学 LangChain — 你会成为"工具型选手"，不是工程师
  • 直接学 Prompt — 缺少底层理解，无法优化和排错
  • 只会调 API — 无法构建可靠的生产系统
  • 只会用现成平台 — 缺乏定制化和问题排查能力

正确的三层学习顺序：

  • 第一层：AI 模型层 — ML → DL → NLP → 微调
  • 第二层：RAG 层 — Embedding → 向量数据库 → Hybrid 检索
  • 第三层：工程层 — 后端 → 数据库 → 消息队列 → Docker → 部署
