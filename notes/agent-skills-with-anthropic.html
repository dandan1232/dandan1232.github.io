agent-skills-with-anthropic 实践笔记

发布于 2026-03-05

核心目标

  • 把 Agent 从“泛化对话”升级为“可交付任务执行器”。
  • 通过 skills 约束能力边界，减少漂移和无效输出。

技能触发策略

  • 优先规则触发：显式命令或明确任务域直接命中技能。
  • 其次语义触发：相似意图匹配到候选技能并排序。
  • 冲突处理：多技能命中时按风险优先级和最小动作原则执行。

上下文治理

  • 短上下文保留任务关键约束，长上下文摘要化持久化。
  • 技能输出必须结构化，便于后续步骤消费。
  • 对高风险任务增加“执行前确认”门禁。

失败兜底

  • 技能不可用时回退到安全回答模板，明确缺失能力。
  • 工具失败时暴露可操作错误信息，不隐藏关键上下文。
  • 保留失败样本，定期用于提示词和路由策略迭代。

结论

技能化的本质是把能力显式化、把流程可控化，最终目标是稳定交付而不是一次性“答得像”。
