大模型应用开发基础：从 LLM 到 RAG、MCP 全景梳理

发布于 2026-05-07 · AI 应用开发

一、什么是大语言模型（LLM）

语言模型的核心任务是预测词序列中下一个词的概率。从统计语言模型到今天的大语言模型，经历了四代演化：

  • 统计语言模型（SLM）：基于马尔可夫假设的 n-gram 模型
  • 神经语言模型（NLM）：基于 RNN 的词嵌入，代表作 word2vec
  • 预训练语言模型（PLM）：基于 Transformer 的双向/单向预训练，代表作 BERT、GPT-1/2
  • 大语言模型（LLM）：基于 Scaling Law 的超大规模模型，具备涌现能力，代表作 GPT-3、ChatGPT、Llama

二、大模型的构建流程

一个大模型的诞生通常经历三个阶段：

2.1 预训练（Pretraining）

使用海量文本数据训练模型参数，目标是为模型找到一个优质的"起点"。当前主流路线是解码器架构 + 预测下一个词。预训练数据规模通常在数 T token 级别（如 Llama-3 使用 15T token），对算力要求极高。

2.2 有监督微调（SFT）

预训练后的模型擅长文本补全，但不擅长遵循人类指令。通过指令微调（Instruction Tuning），用成对的输入-输出数据训练模型以问答形式回答问题，激活其任务解决能力。所需数据量远小于预训练（几千到几十万条即可）。

2.3 人类对齐（RLHF / DPO）

通过基于人类反馈的强化学习（RLHF）使模型输出符合人类价值观。核心是训练一个奖励模型来评判输出质量。简化方案如 DPO（直接偏好优化）可以跳过强化学习，直接用偏好数据优化模型。

三、大模型应用开发范式

3.1 Prompt 工程

通过精心构造提示词直接调教大模型，是最"有手就行"的开发方式。两种关键技术：

  • 上下文学习（ICL）：将任务说明和示例融入提示，利用模型归纳能力完成新任务
  • 思维链提示（CoT）：在提示中引入逻辑推理链条，增强复杂问题的解析能力

3.2 RAG：给 LLM 外接大脑

大模型存在三大局限：知识滞后（不知道训练截止后的事）、数据安全（企业数据不能外传）、幻觉问题（一本正经地胡说八道）。RAG（检索增强生成）通过以下流程解决：

  1. 将知识转成 Embedding 向量，存入向量数据库
  2. 用户提问时，先检索相关知识片段
  3. 将检索结果作为上下文，配合 Prompt 送入 LLM 生成答案

RAG 核心流程详解

  • 文档处理：支持 PDF、PPT、Excel 等多种格式的读取与切分
  • Chunk 分割：如何合理地将文档分割成语义完整的片段
  • Embedding 向量化：选择合适的模型将 Chunks 转为向量
  • 查询改写：对用户提问进行扩充或改写，提高检索召回率
  • 检索与排序：先粗排召回候选，再精排筛选高相关内容
  • Prompt 组装：将检索结果与用户问题组合成完整 Prompt
  • 答案生成与校验：LLM 生成答案后做合规性检查

RAG vs 微调

  • RAG 优势：实时更新知识、高可解释性、适合动态场景
  • 微调优势：定制模型行为、减少幻觉，但需大量计算资源
  • 两者可协同使用，如 RA-DIT 联合优化检索器与生成器

3.3 Embedding 模型

Embedding 模型的任务是将文本转换为向量，使语义相似的文本对应高相似度的向量。训练数据通常是三元组（Anchor、Positive、Negative）。

常用模型对比：OpenAI 的 text-embedding-ada-002 维度 1536，BGE 系列支持中英文，M3E 系列针对中文优化。选择时需考虑维度、速度、多语言支持等因素。

3.4 MCP：大模型的 USB-C 接口

MCP（Model Context Protocol）是 Anthropic 提出的开放标准协议，用于在大模型和数据源之间建立安全双向的连接。可以把它理解为大模型世界的 USB-C——统一了 LLM 访问外部资源的方式。

核心架构

  • MCP Host：包含 MCP Client 的应用（如 Claude Desktop、IDE）
  • MCP Client：使用 MCP 协议与 Server 建立一对一连接
  • MCP Server：连接内外部资源，对外提供服务

Server 提供的三种能力

  • Resources：资源读取（类似文件数据读取）
  • Tools：工具调用（第三方服务、函数）
  • Prompts：预定义的提示词模板

通讯机制

支持两种通讯方式，均使用 JSON-RPC 2.0 格式：

  • 本地通讯（stdio）：Client 启动 Server 作为子进程，通过 stdin/stdout 通信
  • 远程通讯（SSE）：基于 HTTP 的 Server-Sent Events，支持远程部署

Server 代码示例

from mcp.server import Server
import mcp.types as types

server = Server("demo")

@server.list_tools()
async def handle_list_tools():
    return [
        types.Tool(
            name="demo-tool",
            description="Get data tool for a param",
            inputSchema={
                "type": "object",
                "properties": {
                    "param": {"type": "string", "description": "url"}
                },
                "required": ["param"]
            }
        )
    ]

@server.call_tool()
async def handle_call_tool(name, arguments):
    if name == "demo-tool":
        param = arguments.get("param").upper()
        return [types.TextContent(type="text", text=f"text:{param}")]

调试工具

MCP 提供了 MCP Inspector 用于调试和测试 Server：

npx @modelcontextprotocol/inspector
# 启动后访问 http://localhost:5173

四、服务端部署方式

大模型应用的服务端有两种主要方式：

  • 调用 API：便捷、按需付费、无需运维，但有网络延迟和数据隐私风险
  • 本地部署：数据安全、性能可控、可定制，但需要 GPU 投资和运维能力

五、总结

大模型应用开发的核心知识图谱：

  • LLM 基础：理解预训练 → SFT → RLHF 的构建流程
  • Prompt 工程：ICL 和 CoT 是挖掘模型能力的基本功
  • RAG：解决知识滞后和幻觉问题的核心方案
  • Embedding：RAG 的基础设施，决定检索质量
  • MCP：统一 LLM 与外部资源的连接标准

这些技术栈构成了大模型应用开发的基石，掌握它们是进入 AI 应用开发领域的第一步。
