Docker Desktop 数据迁移指南
概述
本指南详细介绍如何将 Docker Desktop 的数据文件从 C 盘迁移到 D 盘，以释放 C 盘空间并优化系统性能。

迁移原理
Docker Desktop 数据存储结构
Docker Desktop 在 Windows 上使用 WSL2 (Windows Subsystem for Linux 2) 作为后端，主要数据存储位置：

C:\Users\{用户名}\AppData\Local\Docker\
├── log/                    # 日志文件
├── run/                    # 运行时文件
├── tasks/                  # 任务配置
└── wsl/                    # WSL2 虚拟磁盘 (主要数据)
    ├── disk/
    │   └── docker_data.vhdx  # Docker 镜像、容器、卷数据 (最大文件)
    └── main/
        └── ext4.vhdx         # WSL2 系统文件
迁移原理
数据移动: 将整个 Docker 数据目录从 C 盘移动到 D 盘
符号链接: 创建从原位置到新位置的符号链接，让 Docker Desktop 无缝访问新位置的数据
透明性: 对 Docker Desktop 来说，数据仍然在原来的路径，但实际存储在 D 盘
准备工作
系统要求
Windows 10/11 with WSL2
Docker Desktop for Windows
管理员权限
目标磁盘有足够空间
检查当前状态
# 检查 Docker 信息
docker info

# 检查数据大小
Get-ChildItem "$env:LOCALAPPDATA\Docker\wsl" -Recurse | Measure-Object -Property Length -Sum | Select-Object @{Name="SizeGB";Expression={[math]::Round($_.Sum/1GB,2)}}, Count

# 检查 D 盘空间
Get-WmiObject -Class Win32_LogicalDisk | Where-Object {$_.DeviceID -eq "D:"} | Select-Object DeviceID, @{Name="SizeGB";Expression={[math]::Round($_.Size/1GB,2)}}, @{Name="FreeSpaceGB";Expression={[math]::Round($_.FreeSpace/1GB,2)}}
迁移步骤
步骤 1: 停止 Docker 服务
重要: 必须完全停止 Docker Desktop 和 WSL 才能安全移动数据文件。

# 停止 Docker Desktop
docker desktop stop

# 关闭所有 WSL 实例
wsl --shutdown

# 确认没有 Docker 进程运行
Get-Process -Name "*Docker*" -ErrorAction SilentlyContinue
原理: VHDX 文件在使用时被独占锁定，必须停止所有相关服务才能移动。

步骤 2: 创建目标目录
# 在 D 盘创建 Docker 目录
New-Item -Path "D:\Docker" -ItemType Directory -Force
步骤 3: 移动数据
# 使用 robocopy 移动数据 (推荐)
robocopy "$env:LOCALAPPDATA\Docker" "D:\Docker" /E /MOVE /R:3 /W:5
参数说明:

/E: 复制子目录，包括空目录
/MOVE: 移动文件和目录（复制后删除源文件）
/R:3: 失败重试 3 次
/W:5: 重试间隔 5 秒
替代方法 (如果 robocopy 失败):

# 手动复制和删除
Copy-Item "$env:LOCALAPPDATA\Docker" "D:\Docker" -Recurse -Force
Remove-Item "$env:LOCALAPPDATA\Docker" -Recurse -Force
步骤 4: 创建符号链接
需要管理员权限，以管理员身份运行 PowerShell：

# 创建符号链接
New-Item -Path "$env:LOCALAPPDATA\Docker" -ItemType SymbolicLink -Value "D:\Docker" -Force
原理: 符号链接是文件系统级别的重定向，当应用程序访问原路径时，系统会自动重定向到新路径。

替代方法 (如果 PowerShell 失败):

# 使用 cmd 命令
mklink /D "C:\Users\{用户名}\AppData\Local\Docker" "D:\Docker"
验证与启动
验证迁移
# 检查符号链接是否创建成功
Get-Item "$env:LOCALAPPDATA\Docker" | Select-Object LinkType, Target

# 检查数据是否在 D 盘
dir "D:\Docker"

# 检查主要数据文件
Get-ChildItem "D:\Docker\wsl\disk" | Select-Object Name, @{Name="SizeGB";Expression={[math]::Round($_.Length/1GB,2)}}
启动 Docker Desktop
# 启动 Docker Desktop
docker desktop start

# 等待启动完成后验证
docker info
docker images
故障排除
常见问题
1. 权限不足
问题: 创建符号链接时提示权限不足

New-Item : 此操作需要管理员权限
解决方案:

以管理员身份运行 PowerShell
或使用开发者模式（Windows 10 1703+）
2. 文件占用
问题: 移动文件时提示文件被占用

robocopy: ERROR 32 (0x00000020) 进程无法访问文件
解决方案:

# 确保完全停止相关服务
Get-Process -Name "*Docker*" | Stop-Process -Force
wsl --shutdown
taskkill /F /IM "Docker Desktop.exe"
3. 符号链接失效
问题: Docker Desktop 启动后找不到数据

解决方案:

# 检查符号链接
Get-Item "$env:LOCALAPPDATA\Docker"

# 重建符号链接
Remove-Item "$env:LOCALAPPDATA\Docker" -Force
New-Item -Path "$env:LOCALAPPDATA\Docker" -ItemType SymbolicLink -Value "D:\Docker" -Force
4. WSL 启动失败
问题: Docker Desktop 启动时 WSL 报错

解决方案:

# 重启 WSL 服务
wsl --shutdown
wsl --unregister docker-desktop
wsl --unregister docker-desktop-data

# 重新启动 Docker Desktop，会自动重建 WSL 分发
恢复原状
如果需要回滚迁移：

# 停止 Docker
docker desktop stop
wsl --shutdown

# 删除符号链接
Remove-Item "$env:LOCALAPPDATA\Docker"

# 移回数据
robocopy "D:\Docker" "$env:LOCALAPPDATA\Docker" /E /MOVE

# 删除 D 盘目录
Remove-Item "D:\Docker" -Recurse -Force

磁盘类型: SSD 性能优于 HDD
磁盘空间: 保留至少 20% 空间用于临时文件
碎片整理: 定期整理 VHDX 文件
维护建议
定期备份: 备份 D:\Docker 目录
监控空间: 定期检查磁盘使用情况
清理镜像: 定期清理不用的 Docker 镜像
# 清理命令
docker system prune -a
docker volume prune
安全考虑
权限设置: 确保只有当前用户可访问 Docker 数据
备份加密: 备份时考虑加密敏感数据
访问控制: 限制对 Docker 数据目录的访问
高级配置
Docker Desktop 设置
可以通过修改 Docker Desktop 配置文件来自定义更多设置：

// C:\Users\{用户名}\AppData\Roaming\Docker\settings-store.json
{
  "dataRoot": "D:\\Docker\\data",  // 自定义数据根目录
  "autoStart": false,              // 禁用自动启动
  "memoryMiB": 4096               // 限制内存使用
}
WSL2 优化
# WSL2 配置文件: C:\Users\{用户名}\.wslconfig
[wsl2]
memory=4GB
processors=4
swap=0
localhostForwarding=true
总结
迁移后的目录结构：

C:\Users\{用户名}\AppData\Local\Docker -> D:\Docker (符号链接)
D:\Docker\                                        (实际数据)
├── wsl\disk\docker_data.vhdx                    (主要数据 ~7.8GB)
├── wsl\main\ext4.vhdx                           (WSL 系统 ~120MB)
└── 其他配置和日志文件
